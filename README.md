The application uses a central USERS table to manage corporate identities. Below is the structure generated by the JPA Entity:

SQL

CREATE TABLE USERS (
    ID NUMBER(19,0) PRIMARY KEY,
    FULL_NAME VARCHAR2(255) NOT NULL,
    EMAIL VARCHAR2(255) UNIQUE NOT NULL,
    PASSWORD VARCHAR2(255),       -- Hashed with BCrypt
    PROVIDER VARCHAR2(50),       -- 'LOCAL' or 'GOOGLE'
    PHOTO_URL VARCHAR2(500),     -- Links to Google or UI-Avatars
    CREATED_AT TIMESTAMP
);

üõ†Ô∏è Security & Logic Flow
Password Security: Uses BCryptPasswordEncoder to ensure zero plain-text passwords are stored in the database.

OAuth2 Flow: When a user logs in via Google, the system checks the EMAIL field. If the user exists, it updates the PHOTO_URL; if not, it creates a new record with the GOOGLE provider.

Safe Deletion: The delete process requires a POST request. For security, the system clears the SecurityContextHolder after the database transaction is complete to prevent "Ghost Sessions."
